# 00 - Introduction

## How to Use This Repository

This is a structured, practical DevOps learning path. Each numbered folder is a topic with:

- **README.md** — learning objectives, checklist, and a hands-on project
- **notes/** — your notes as you learn (write things down, it helps retention)
- **projects/** — your project code for that topic

## Rules for Yourself

1. **Don't skip the project.** Reading and watching is not learning. Building is learning.
2. **Don't move on until you pass the validation.** Each section has a "you're done when" checklist.
3. **Break things on purpose.** The best learning happens when you break something and fix it.
4. **Document as you go.** Future you will thank present you.
5. **Use the real tools.** Local VMs, real AWS accounts (free tier), real clusters. Not just tutorials.

## Prerequisites

- A computer with at least 16GB RAM (for running local K8s clusters, Docker, VMs)
- An AWS account (free tier covers most of what you'll need)
- A GitHub account
- A cheap domain name (~$10/year) for DNS and TLS exercises
- Comfort with at least one programming language

## Estimated Timeline

- Full-time (40 hrs/week): ~9 months
- Part-time (15 hrs/week): ~18 months
- Weekends only (8 hrs/week): ~2+ years

Adjust based on your pace. Speed doesn't matter. Depth does.
